# Glossary

This glossary provides definitions for common terms related to Avalanche.

## A

*   **Avalanche-CLI:** A command-line tool for managing local testnets, keys, and wallets.
*   **AvalancheGo:** The official implementation of the Avalanche protocol.
*   **AVAX:** The native token of the Avalanche network.

## B

*   **Bridge:** A tool for transferring assets between different blockchain networks.

## C

*   **C-Chain:** The contract chain, which is used for creating and executing smart contracts.
*   **Consensus:** The process by which a set of nodes in a network agree on the state of the network.

## D

*   **dApp:** A decentralized application.

## E

*   **EVM:** The Ethereum Virtual Machine.

## F

*   **Fork:** A change to the protocol of a blockchain network.

## G

*   **Gas:** A unit of measurement for the computational resources that are required to execute a transaction.
*   **Genesis file:** A file that defines the initial state of a blockchain network.
*   **Governance:** The process of managing a blockchain network.

## H

*   **Hard fork:** A backward-incompatible upgrade to a blockchain network.

## I

*   **Interoperability:** The ability of different blockchain networks to communicate with each other.

## K

*   **Kubernetes:** A container orchestration platform.

## L

*   **Load balancing:** The process of distributing traffic across multiple nodes in a network.

## M

*   **Mainnet:** The main network of a blockchain.

## N

*   **Node:** A computer that is connected to a blockchain network.

## P

*   **P-Chain:** The platform chain, which is responsible for coordinating validators and managing Subnets.
*   **Primary Network:** The main network of the Avalanche network, which is composed of the C-Chain, P-Chain, and X-Chain.

## S

*   **Scalability:** The ability of a blockchain network to handle a large number of transactions.
*   **Smart contract:** A program that is stored on a blockchain.
*   **Snowman:** The consensus protocol that is used in the Avalanche network.
*   **Soft fork:** A backward-compatible upgrade to a blockchain network.
*   **Staking:** The process of locking up a certain amount of cryptocurrency in order to participate in the consensus process.
*   **Subnet:** A custom blockchain network that is created by developers to support their specific use cases.

## T

*   **Testnet:** A separate blockchain that is used for testing.
*   **Tokenomics:** The study of the economic properties of a cryptocurrency.

## V

*   **Validator:** A node that is responsible for securing a blockchain network and processing transactions.
*   **VM:** A virtual machine.

## W

*   **Wallet:** A program that is used to store and manage cryptocurrency.

## X

*   **X-Chain:** The exchange chain, which is used for creating and exchanging assets.
